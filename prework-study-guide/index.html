HTML

  • The head element contains information about the webpage.
  • The information that the head element contains is called, "Metadata". Metadata contains unviewable information about the webpage in the form of metadata tags.
  • The Metadata tags that contain important information are as follows:
    1. the "title" element which defines the title for a webpage
    2. the "style" element which defines the CSS style for a page
    3. the "base" element which defines the base URL for a page
    4. the "link" element which connects any external resources to the current HTML document
    5. the "meta" element which defines any extra metadata like character set, the page author, keywords, etc;
    6. the "script" element which embeds scripts in the HTML document
  • The body element represents the visible content shown to the user.
  • HTML also includes, "HTML attributes" which are phrases that provide additional improtant information about the page.
  • HTML attributes are always defined in the beginning element.

CSS

  • There are 3 ways to style a webpage with CSS, incline CSS, internal CSS style sheet, and external CSS style sheet. External CSS style sheet is the most common.
  • CSS styling uses "CSS rules" which groups CSS properties that will be applied to the HTML elements.
  • Each CSS rule includes a selector (which defines the element that the rules will be applied to) and a declaration (comprised of 2 important CSS components: the desired CSS property we want to apply and the value of that property) or declaration block for multiple declarations.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A border creates a border around the element.
  • A box shadow adds a shadow effect around the element's frame
  • A Class Attribute allows the user to share a specific CSS rule to any element of their choosing, by way of a class selector.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin (master or main): downloads new commits from the remote repository and immediately updates the local repository with those changes
  • git clone: copies the current repository and moves the copy in a new directory, usually in a new location
  • git add: adds a change in the current working directory to the staging area, or what the user wants to add to the next commit
  • git commit: saves the file and leaves a snapshot or checkpoint for the user to refer back to; tip: it's a good idea to use "git commit -m" to add a commit message stating what was changed

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A condition or conditional statement, is a type of decision-making piece of code. It uses values such as the if statement to determine if code needs to be executed (if true, then yes and the control flow continues; if false, then no and the control flow stops).
  • A loop allows for code that is similar or repitive to be repeated until the user wishes for it to stop (usually when a certain value is reached).
  • An array is a single variable that holds a group of data that is usually synonymous in some way, shape, or form.
  • A function is a set of instructions that directs the computer on how to complete a specific task. It does not automatically run when the file does. It must be manually called by name.
